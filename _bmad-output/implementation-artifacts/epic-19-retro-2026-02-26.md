## What Went Well

1. **Rapid execution** — All 4 stories completed in a single day. Efficient scaffolding from purchased layout with clean adaptation into own codebase.

2. **Pragmatic auth decision** — Originally specced as username/password with env vars. Team chose `x-api-key` (same as backend) — simpler, sufficient for single-user/dev. AWS Cognito migration properly deferred to Phase 9 (B-33). Decision documented in epics.md with rationale.

3. **Repository hygiene** — Purchased layout reference (`web_interface_target/`) removed from repo (commit e8a44fd). Design incorporated into `web_interface_app2/`, no external dependencies left.

4. **Quick security fix** — Hardcoded API key detected and removed (commit 5c57356). API key now provided via login page.

5. **Infrastructure reuse** — Leveraged existing CloudFront/S3 stacks and SSM-based deploy scripts (B-51) for seamless deployment.

## What Could Be Better

1. **Hardcoded API key committed to repo** — A real API key was committed in 478b62c and only removed in 5c57356. Even though fixed quickly, the key exists in git history. This is a serious security concern.

2. **Pre-commit hooks not working at time of commit** — TruffleHog pre-commit was misconfigured and did not catch the secret. Since fixed, and a second secret detection tool added (defense in depth).

3. **Zero story files in implementation-artifacts** — Third consecutive epic with missing story documentation (after 17-4, 18-consolidation). No Dev Agent Records, no debug logs, no file lists.

4. **No documented code review** — None of the 4 stories show evidence of code review. For an epic involving authentication, this is a gap.

5. **Speed vs process trade-off** — Completing everything in one day meant skipping formal checkpoints. For security-sensitive work (auth), this is risky.

## Key Learnings

- **Speed does not exempt from security discipline** — Secrets in repo are a risk regardless of development pace. Pre-commit hooks are the safety net, and they must work.
- **Defense in depth for secret detection** — One tool can have blind spots. Two complementary tools significantly reduce risk of secrets reaching the repository.
- **Pragmatism with documentation** — Simplifying auth was correct, and documenting the rationale in epics.md preserved context for future developers.
- **Story files rule needs formal revision** — Three epics in a row breaking the same rule means the rule doesn't match reality. Time to formalize when story files are required vs optional.
